# microCMS リッチエディタ 入力マニュアル

**対象**: ストーリー（`content` フィールド）・物件詳細（`description` フィールド）

このマニュアルでは、Figmaデザインに登場する各要素を、microCMSのリッチエディタ上でどのように入力するかを説明します。

## 1. 全体の構成イメージ

1つの記事は、複数の「セクション」で構成されます。各セクションは以下の流れで入力します：

```
[区切り線] ← 2つ目のセクション以降
[セクションカテゴリ]  ← h5: 例）「この家との出会い」
[セクション見出し]    ← h3: 例）「この家に、静けさとあたたかさがあった。」
[本文テキスト]        ← 段落（p）
[画像]                ← 画像挿入
[画像キャプション]    ← イタリック（斜体）テキスト
[コメント吹き出し]    ← 引用（blockquote）
```

> **ポイント**: 左側のタイムライン型「目次ナビ」は、入力した見出し（h5）から自動的に生成されるため、入力不要です。

---

## 2. セクションカテゴリ（小見出し）

デザイン上、セクション見出しの上に表示される短いラベルテキストです。

**デザイン例**: 「この家との出会い」「〇〇の魅力」「周辺環境」「担当者より」

### 入力方法

1. テキストを入力する
2. ツールバーの「見出し」ドロップダウンから **「見出し5（H5）」** を選択する

### 出力されるHTML

```html
<h5>この家との出会い</h5>
```

### 注意

- **見出し3（H3）と間違えないよう注意してください。** カテゴリラベルは必ず **H5** です。
- 1〜3文字程度の短いテキストを入れてください
- このテキストが目次ナビの項目として自動表示されます

---

## 3. セクション見出し（大見出し）

各セクションのメインタイトルです。デザイン上は明朝体で大きく表示されます。

**デザイン例**: 「この家に、静けさとあたたかさがあった。」「暮らしの目線で、家の中を歩いてみる。」

### 入力方法

1. テキストを入力する
2. ツールバーの「見出し」ドロップダウンから **「見出し3（H3）」** を選択する

### 出力されるHTML

```html
<h3>この家に、静けさとあたたかさがあった。</h3>
```

### 注意

- セクション見出しは必ず **H3** を使ってください
- H1・H2はページ全体のタイトル用に予約されているため、記事本文内では使わないでください

---

## 4. 本文テキスト

記事のメインとなる文章です。デザイン上はゴシック体で表示されます。

**デザイン例**: 「静かな住宅街の一角。地元・三島市谷田に建つこの家を初めて訪れたのは、春の昼下がりでした。…」

### 入力方法

1. そのまま文章を入力する（特別な操作は不要）
2. 改行で段落を分ける

### 出力されるHTML

```html
<p>静かな住宅街の一角。地元・三島市谷田に建つこの家を...</p>
<p>私たちスタッフも、案内のたびに...</p>
```

### 注意

- 段落を分けたいところでは **Enterキー** を押してください（空行が入り、段落が分かれます）
- 同じ段落内で改行したい場合は **Shift + Enter** を押してください
- 太字や文字色などの装飾は基本的に使わないでください（デザインの統一性が崩れます）

---

## 5. 画像の挿入

セクション内に表示する写真です。デザイン上は角丸で横幅いっぱいに表示されます。

**デザイン例**: 物件外観の写真、室内の写真、周辺環境の写真など

### 入力方法

1. 画像を挿入したい位置にカーソルを置く
2. ツールバーの **「画像」アイコン** をクリック
3. 画像をアップロードまたはメディアライブラリから選択

### 出力されるHTML

```html
<img src="https://images.microcms-assets.io/..." alt="" />
```

### 注意

- 画像は**横長（3:2程度のアスペクト比）**で用意してください
- 推奨サイズ: **横幅 1600px 以上**（高解像度ディスプレイ対応）
- 画像は自動的に横幅いっぱい、角丸24pxで表示されます

---

## 6. 画像キャプション

画像の下に表示される説明テキストです。

**デザイン例**: 「南向きの外観」「担当者が物件を案内している様子」

### 入力方法

1. 画像を挿入した **直後の行** にキャプションテキストを入力する
2. テキストを選択し、ツールバーの **「イタリック（斜体）」ボタン（I）** をクリックする

### 出力されるHTML

```html
<img src="..." alt="" />
<p><em>南向きの外観</em></p>
```

### 注意

- **必ずイタリック（斜体）にしてください。** サイト側はイタリックのテキストをキャプションとして認識し、小さめのフォントサイズ・左寄せで表示します
- キャプションが不要な画像の場合は、イタリックの行を入れなくてOKです
- キャプションは1行に収まる短いテキストにしてください

---

## 7. 複数画像の並べ方

1つのセクションに複数の写真を載せたい場合の入力方法です。

**デザイン例**: 「〇〇の魅力」セクションで室内写真を2〜3枚連続で表示

### 入力方法

1. 1枚目の画像を挿入
2. キャプション（イタリック）を入力（任意）
3. 2枚目の画像を挿入
4. キャプション（イタリック）を入力（任意）
5. …繰り返し

### リッチエディタ上のイメージ

```
[画像1]
南向きの外観            ← イタリック
[画像2]
玄関ホールの様子        ← イタリック
[画像3]
```

### 注意

- 画像は **縦に並べて** 入力してください（横並びレイアウトはサイト側で自動処理されます）
- 1セクションあたり **5枚程度まで** を目安にしてください

## 8. コメント吹き出し（引用ブロック）

担当者からの一言コメントや補足情報を、吹き出し風のデザインで表示する要素です。

**デザイン例**: アイ企画のアイコン + 薄緑の吹き出し「ここに一言コメントが入ります」

### 入力方法

1. コメントテキストを入力する
2. テキストを選択する
3. ツールバーの **「引用」ボタン（❝）** をクリックする

### 出力されるHTML

```html
<blockquote><p>ここに一言コメントが入ります</p></blockquote>
```

### 注意

- 引用ブロックはサイト上で自動的に「アイ企画のアイコン + 薄緑の吹き出し」として表示されます
- 1〜3文程度の短いコメントを入れてください
- 引用ブロック内には **画像を入れないでください**

---

## 9. セクション間の区切り線

セクションとセクションの間を視覚的に区切る水平線です。

### 入力方法

1. セクションの終わり（次のカテゴリラベルの前）にカーソルを置く
2. ツールバーの **「水平線」ボタン（―）** をクリック

### 出力されるHTML

```html
<hr />
```

### 注意

- **最初のセクションの前には入れないでください**（2つ目のセクションから挿入します）
- 記事の最後にも入れる必要はありません

---

## 10. 入力しなくてよい要素

以下の要素はリッチエディタでは入力不要です。サイト側で自動的に表示・生成されます。

| 要素 | 理由 |
|------|------|
| **ヒーロー画像**（記事上部の大きな写真） | サムネイル画像フィールドから自動表示 |
| **タイトル** | タイトルフィールドから自動表示 |
| **サブタイトル / 冒頭説明文** | サブタイトルフィールドから自動表示 |
| **目次ナビ（左サイドバー）** | H5見出しから自動生成 |
| **タグ・地域ラベル** | カテゴリ・地域フィールドから自動表示 |
| **関連物件リンクカード**（ストーリー） | 物件リレーションフィールドから自動表示 |
| **物件概要テーブル**（物件詳細） | 各構造化フィールド（所在地・間取り等）から自動表示 |
| **物件資料・お問い合わせボタン**（物件詳細） | 自動表示 |
| **関連ストーリー / 関連物件**（記事下部） | 自動表示 |
| **パンくずリスト** | 自動生成 |

## 11. 対応表まとめ

| デザイン要素 | リッチエディタ操作 | HTMLタグ | 例 |
|---|---|---|---|
| セクションカテゴリ | 見出し5（H5） | `<h5>` | この家との出会い |
| セクション見出し | 見出し3（H3） | `<h3>` | この家に、静けさとあたたかさがあった。 |
| 本文テキスト | そのまま入力 | `<p>` | 静かな住宅街の一角。… |
| 画像 | 画像挿入 | `<img>` | （写真） |
| 画像キャプション | イタリック（斜体） | `<p><em>` | 南向きの外観 |
| コメント吹き出し | 引用 | `<blockquote>` | ここに一言コメントが入ります |
| セクション区切り | 水平線 | `<hr>` | ― |

## 12. 入力例（ストーリー）

以下は、ストーリー記事の `content` フィールドに入力するリッチエディタの内容例です。

```

[H5] この家との出会い

[H3] この家に、静けさとあたたかさがあった。

静かな住宅街の一角。
地元・三島市谷田に建つこの家を初めて訪れたのは、
春の昼下がりでした。やわらかな光が落ちる南向きの庭と、
どこか落ち着いた佇まい。

私たちスタッフも、案内のたびに
「ここに暮らす人は、きっとこんなふうに過ごすんだろうな」
と思い描いてしまう、そんな家です。

[画像: property-exterior.jpg]
南向きの外観                          ← イタリック

  ❝ 初めて見たとき、家の空気感が     ← 引用ブロック
  　 とても穏やかでした。             ← 引用ブロック

────── 水平線 ──────

[H5] 〇〇の魅力

[H3] 暮らしの目線で、家の中を歩いてみる。

玄関を開けると、廊下が一直線に伸び、
```

## 13. 入力例（物件詳細）

物件詳細ページの `description` フィールドもストーリーと同じ書き方です。
ただし、物件概要テーブル・価格・ボタン等は構造化フィールドで入力済みのため、リッチエディタには「物件の紹介文」のみを記載します。

```
━━━ リッチエディタ画面イメージ ━━━

[H5] この家との出会い

[H3] この家に、静けさとあたたかさがあった。

静かな住宅街の一角に佇む、
ちょっと個性的な住まい。コンパクトながら、
南側にひらけた玄関前の庭では
季節の花や家庭菜園も楽しめます。

[画像: exterior.jpg]
物件の外観                            ← イタリック

────── 水平線 ──────

[H5] 物件の魅力

[H3] 暮らしの余白を、自分好みに。

昭和50年築の鉄骨造2階建。
しっかりした構造で、まだまだ活かせる
昭和の良き時代のつくり。

[画像: interior.jpg]
和室から庭を望む                      ← イタリック

  ❝ リフォーム前提だからこそ、       ← 引用ブロック
  　 自由度が高い物件です。           ← 引用ブロック

（以下、同様のパターンで続く）

━━━━━━━━━━━━━━━━━━━━━━━
```

> **物件詳細の場合のポイント**: 所在地・面積・間取り・築年月・価格などの情報はリッチエディタには書かず、それぞれ専用のフィールドに入力してください。リッチエディタには「この物件のストーリー・魅力を伝える文章」のみを入力します。

## よくある質問

### Q. 見出しレベルを間違えてしまった
テキストを選択し直して、正しい見出しレベル（H5 or H3）を選び直せばOKです。

### Q. 太字や文字色は使ってよいか？
基本的には使わないでください。デザインの一貫性が崩れる原因になります。強調したい場合でも、文章の書き方で伝えるようにしてください。

### Q. リスト（箇条書き）は使ってよいか？
デザイン上はリストのパターンがありませんが、本文中に情報を整理したい場合は使用可能です。ただし多用は避けてください。

### Q. テーブル（表）は使ってよいか？
物件詳細の仕様情報は専用フィールドで入力するため、リッチエディタ内でテーブルを使う必要はありません。

### Q. 画像のサイズや比率は？
横長（3:2 程度）の画像を推奨します。縦長画像はレイアウトが崩れる場合があります。横幅1600px以上の画像を使用してください。
